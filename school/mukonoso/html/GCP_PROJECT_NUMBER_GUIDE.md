# GCPプロジェクト番号の設定ガイド

Google Cloud Platform（GCP）プロジェクト番号についての説明と推奨設定をまとめます。

---

## 🔍 プロジェクト番号とは

### プロジェクト番号の特徴

- **自動生成**: Googleが自動的に割り当てる数字（例：`123456789012`）
- **一意性**: 各プロジェクトに固有の番号が割り当てられます
- **変更不可**: 一度割り当てられた番号は変更できません
- **手動設定不可**: ユーザーが自由に設定することはできません

### プロジェクトIDとの違い

- **プロジェクトID**: ユーザーが設定できる名前（例：`iteen-reservation-management`）
- **プロジェクト番号**: Googleが自動的に割り当てる数字（例：`123456789012`）

---

## ✅ 推奨設定

### 既存のプロジェクトがある場合

**推奨: 既存のプロジェクトを使用**

1. **Google Apps Scriptのプロジェクト設定を確認**
   - 左側のメニューから「プロジェクトの設定」（⚙️アイコン）をクリック
   - 「Google Cloud Platform（GCP）プロジェクト」セクションを確認
   - プロジェクトIDが表示されている場合は、そのまま使用できます

2. **プロジェクト番号を確認**
   - プロジェクトIDの下にプロジェクト番号が表示されます
   - この番号をそのまま使用してください

**利点:**
- 既存の設定を維持できる
- 追加の設定が不要
- 権限の再設定が不要

### 新しいプロジェクトを作成する場合

**推奨: 新しいプロジェクトを作成**

1. **Google Apps Scriptのプロジェクト設定を開く**
   - 左側のメニューから「プロジェクトの設定」（⚙️アイコン）をクリック
   - 「Google Cloud Platform（GCP）プロジェクト」セクションで「プロジェクト番号を変更」をクリック

2. **新しいプロジェクトを作成**
   - 「こちら」リンクをクリックしてGoogle Cloud Consoleを開く
   - 「プロジェクトを作成」をクリック
   - 以下の情報を入力：
     - **プロジェクト名**: `iTee予約管理` または `iteen-reservation-management`
     - **プロジェクトID**: 自動生成されるか、手動で設定（例：`iteen-reservation-management`）
       - 小文字、数字、ハイフンのみ使用可能
       - グローバルに一意である必要がある
   - 「作成」をクリック

3. **プロジェクト番号を確認**
   - プロジェクト作成後、Google Cloud Consoleでプロジェクト番号を確認
   - プロジェクト番号は自動的に割り当てられます（例：`123456789012`）

4. **Google Apps Scriptに戻る**
   - Google Apps Scriptのプロジェクト設定画面に戻る
   - 「GCPのプロジェクト番号」フィールドに、自動的に割り当てられたプロジェクト番号を入力
   - 「プロジェクトを設定」ボタンをクリック

**注意:**
- プロジェクト番号は自動的に割り当てられるため、手動で設定する必要はありません
- プロジェクト番号を入力する際は、Google Cloud Consoleで確認した番号をそのまま使用してください

## 📋 プロジェクト名の推奨設定

### 推奨プロジェクト名

1. **日本語名**: `iTee予約管理`
   - わかりやすい日本語名
   - Google Cloud Consoleで表示される名前

2. **プロジェクトID**: `iteen-reservation-management`
   - 小文字、数字、ハイフンのみ使用
   - グローバルに一意である必要がある
   - わかりやすい名前

### プロジェクトIDの命名規則

- **使用可能な文字**: 小文字（a-z）、数字（0-9）、ハイフン（-）
- **使用不可な文字**: 大文字、アンダースコア、スペース、特殊文字
- **長さ**: 6文字以上30文字以下
- **一意性**: グローバルに一意である必要がある

**例:**
- ✅ `iteen-reservation-management`
- ✅ `iteen-mukonoso-reservation`
- ❌ `iTee予約管理`（大文字と日本語が含まれている）
- ❌ `iteen_reservation`（アンダースコアが含まれている）

## ⚠️ 重要な注意事項

### プロジェクトを変更する場合

1. **古いプロジェクトの認証が取り消される**
   - すべてのユーザー認証が完全に取り消されます
   - 再認証が必要になる場合があります

2. **Apps Scriptで管理するプロジェクトに戻せない**
   - 一度変更すると、元のプロジェクトに戻すことができません
   - 慎重に判断してください

3. **新しいプロジェクトでAPIを有効化する必要がある**
   - 新しいプロジェクトでは、必要なAPIを再度有効化する必要があります
   - 例：Cloud Logging API、Google Drive API、Google Sheets APIなど

## 💡 よくある質問

### Q1: プロジェクト番号を自分で設定できますか？

**A:** いいえ、プロジェクト番号はGoogleが自動的に割り当てるため、手動で設定することはできません。プロジェクト作成後、自動的に割り当てられた番号を使用してください。

### Q2: 既存のプロジェクトと新しいプロジェクト、どちらを使うべきですか？

**A:** 既存のプロジェクトがある場合は、そのまま使用することを推奨します。新しいプロジェクトを作成すると、認証の再設定やAPIの再有効化が必要になるためです。

### Q3: プロジェクト番号はどこで確認できますか？

**A:** 以下の方法で確認できます：
1. Google Apps Scriptのプロジェクト設定画面
2. Google Cloud Consoleのプロジェクト選択画面
3. Google Cloud Consoleのプロジェクト設定画面
