# ログエクスプローラーへのアクセス方法

Google Cloud Consoleでログエクスプローラーにアクセスする方法を説明します。

---

## 📍 現在の画面からログエクスプローラーにアクセスする方法

### 手順1: 左側のナビゲーションメニューを確認

現在の画面（API / サービスの詳細）では、左側のナビゲーションメニューに以下のセクションが表示されています：

- **API とサービス**（現在選択中）
  - 有効な API とサービス
  - ライブラリ
  - 認証情報
  - OAuth 同意画面
  - ページの使用に関する契約

### 手順2: 「ロギング」セクションに移動

1. **左側のナビゲーションメニューをスクロール**
   - 現在は「API とサービス」セクションが選択されています
   - メニューを下にスクロールして「ロギング」セクションを探す

2. **「ロギング」をクリック**
   - 左側のメニューで「ロギング」をクリック
   - 「ロギング」セクションが展開されます

3. **「ログエクスプローラー」をクリック**
   - 「ロギング」セクションを展開すると、以下のサブメニューが表示されます：
     - **ログエクスプローラー**（Logs Explorer）← これをクリック
     - ログベースの指標（Log-based metrics）
     - エクスポート（Exports）
     - シンク（Sinks）
     - など

---

## 🔍 ログエクスプローラーの使い方

### ログをフィルタする方法

1. **リソースタイプでフィルタ**
   - 「リソースタイプ」ドロップダウンから「Google Apps Script」を選択

2. **時間範囲を設定**
   - 画面右上の時間範囲セレクターで期間を選択
   - 例：過去1時間、過去24時間、過去7日間など

3. **クエリを実行**
   - 「クエリを実行」ボタンをクリック
   - または、Enterキーを押す

### ログの確認

- `console.log()`で出力した内容が表示されます
- 各ログエントリには、タイムスタンプ、ログレベル、メッセージなどが含まれます
- ログをクリックすると、詳細情報が表示されます

---

## 📋 直接URLでアクセスする方法

ログエクスプローラーに直接アクセスするURL：

```
https://console.cloud.google.com/logs/query?project=YOUR_PROJECT_ID
```

`YOUR_PROJECT_ID`を実際のプロジェクトIDに置き換えてください。

**例:**
```
https://console.cloud.google.com/logs/query?project=secure-analyzer-487700-i3
```

## 💡 よくある質問

### Q1: 「ロギング」セクションが見つからない

**A:** 以下の点を確認してください：

1. **左側のメニューをスクロール**
   - メニューが長い場合、下にスクロールする必要があります

2. **メニューが折りたたまれている**
   - ハンバーガーメニュー（☰）をクリックしてメニューを展開

3. **権限の問題**
   - ログエクスプローラーにアクセスするには、適切な権限が必要です
   - プロジェクトのオーナーまたは編集者権限が必要です

### Q2: ログが表示されない

**A:** 以下の点を確認してください：

1. **リソースタイプを確認**
   - 「リソースタイプ」で「Google Apps Script」が選択されているか確認

2. **時間範囲を確認**
   - 時間範囲が適切に設定されているか確認
   - 過去の期間を選択してみる

3. **ログが生成されているか確認**
   - 本番環境で予約フォームを送信
   - `console.log()`が実行されているか確認

### Q3: ログエクスプローラーと実行数画面のログの違いは？

**A:** 

- **実行数画面のログ**: `Logger.log()`と`console.log()`の両方が表示される、簡単に確認できる
- **ログエクスプローラー**: `console.log()`のみが表示される、詳細な分析が可能

通常のデバッグには、実行数画面でログを確認すれば十分です。
