# Google Apps Scriptのログ確認方法

クライアント側（ブラウザ）のログは取れているのに、Google Apps Script側のログが確認できない場合の対処法を説明します。

---

## ⚠️ 重要なポイント

### `no-cors`モードの制限

現在、`reserve.html`では`no-cors`モードでリクエストを送信しています：

```javascript
fetch(CONFIG.GOOGLE_APPS_SCRIPT_URL, {
  method: 'POST',
  mode: 'no-cors',  // ← これが原因
  headers: {
    'Content-Type': 'application/json',
  },
  body: JSON.stringify(emailData)
})
```

**`no-cors`モードの制限:**
- ✅ リクエストはGoogle Apps Scriptに到達します
- ❌ レスポンスを取得できません
- ❌ クライアント側では成功/失敗が分かりません

**そのため、Google Apps Script側のログを確認する必要があります。**

---

## 🔍 Google Apps Script側のログを確認する手順

### 手順1: Google Apps Scriptのエディタを開く

1. **Google Apps Scriptにアクセス**
   - https://script.google.com/ にアクセス
   - `iteen.mukonosou@gmail.com`でログイン

2. **プロジェクトを開く**
   - プロジェクト「iTee予約管理」を開く

### 手順2: 実行数画面を開く

1. **「実行数」をクリック**
   - 左側のメニューから「実行数」アイコン（三本線と再生ボタンのアイコン）をクリック

2. **実行履歴を確認**
   - 画面中央に実行履歴のテーブルが表示されます
   - 最新の`doPost`実行を探します

### 手順3: 個別の実行をクリックしてログを確認

#### 方法1: 実行履歴の行をクリック（通常の方法）

1. **実行履歴の行をクリック**
   - 確認したい実行の行をクリックします
   - 例：最新の`doPost`実行（12:22:16）をクリック

2. **ログパネルを確認**
   - 画面下部または右側にログパネルが表示されます
   - `Logger.log()`や`console.log()`で出力した内容が表示されます

**⚠️ ログパネルが表示されない場合：**
- 画面をスクロールして、下部や右側にログパネルがないか確認
- ブラウザの画面サイズを調整してみる
- 別の方法（方法2または方法3）を試す

#### 方法2: 実行履歴の右端のメニューから開く

1. **実行履歴の行の右端を確認**
   - 各行の右端に「⋮」（三点リーダー）アイコンがあります
   - 青くハイライトされている行（12:22:16の`doPost`実行）の右端を確認

2. **メニューを開く**
   - 「⋮」アイコンをクリック
   - メニューから「ログを表示」や「詳細を表示」を選択

#### 方法3: コードエディタ画面からログを確認（推奨：確実に表示される）

1. **コードエディタ画面に戻る**
   - 左側のメニューから「コード」アイコン（</>）をクリック
   - または、上部の「コード」タブをクリック

2. **「表示」メニューからログを開く**
   - メニューバーの「表示」をクリック
   - 「ログ」を選択
   - または、キーボードショートカット：`Ctrl+Enter`（Windows）/ `Cmd+Enter`（Mac）

3. **ログパネルを確認**
   - 画面下部にログパネルが表示されます
   - 最新の実行ログが自動的に表示されます

**この方法の利点：**
- 確実にログパネルが表示される
- 実行履歴をクリックしなくても、最新のログが表示される
- ログパネルが常に表示されるため、確認しやすい

## 📋 確認すべきログの内容

### 1. `doPost`関数が呼び出されているか

以下のログが表示されているか確認：

```
=== doPost関数が呼び出されました ===
📥 受信データ: {...}
📋 フォームタイプ判定: 予約
📅 予約フォームの処理を開始
```

### 2. `saveToGoogleSheets`関数が呼び出されているか

以下のログが表示されているか確認：

```
📊 Google Sheetsへの保存を開始します...
📊 saveToGoogleSheets関数が呼び出されました
📊 タイプ: reservation
📊 データ: {...}
```

### 3. スプレッドシートが作成/開かれているか

**初回実行の場合:**
```
📝 初回実行のため、新しいスプレッドシートを作成します...
✅ 新規スプレッドシートを作成しました
🔑 スプレッドシートID: XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
📄 スプレッドシートURL: https://docs.google.com/spreadsheets/d/XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX/edit
```

**2回目以降の場合:**
```
📂 既存のスプレッドシートを開こうとしています...
🔑 スプレッドシートID: XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
✅ 既存のスプレッドシートを開きました: iTeen 武庫之荘校 - メール送信履歴
📄 スプレッドシートURL: https://docs.google.com/spreadsheets/d/XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX/edit
```

### 4. データが書き込まれているか

以下のログが表示されているか確認：

```
📝 データを追加します...
✅ データを追加しました
✅ Google Sheetsに履歴を保存しました。結果: success
```

## ❌ ログが表示されない場合の対処法

### 問題1: `doPost`関数のログが表示されない

**原因:**
- リクエストがGoogle Apps Scriptに到達していない
- デプロイが正しく行われていない

**対処法:**
1. **デプロイを確認**
   - 「デプロイ」→「管理デプロイ」を開く
   - 最新のバージョンがデプロイされているか確認

2. **URLを確認**
   - `reserve.html`の`GOOGLE_APPS_SCRIPT_URL`が正しいか確認
   - デプロイURLと一致しているか確認

3. **ブラウザのコンソールを確認**
   - エラーメッセージが表示されていないか確認
   - 「リクエストを送信しました (no-corsモード)」が表示されているか確認

### 問題2: `saveToGoogleSheets`関数のログが表示されない

**原因:**
- `handleReservationForm`関数内でエラーが発生している
- `saveToGoogleSheets`関数が呼び出されていない

**対処法:**
1. **エラーログを確認**
   - 実行履歴で「エラー」ステータスの実行を探す
   - エラーメッセージを確認

2. **`handleReservationForm`関数のログを確認**
   - 「📅 予約フォームの処理を開始」が表示されているか確認
   - その後のログが表示されているか確認

### 問題3: スプレッドシートが作成されない

**原因:**
- `SpreadsheetApp.create()`が失敗している
- 権限が不足している

**対処法:**
1. **エラーログを確認**
   - 「❌ スプレッドシート作成エラー」が表示されていないか確認
   - エラーメッセージを確認

2. **権限を確認**
   - Google Apps Scriptの実行権限が付与されているか確認
   - 初回実行時に権限を付与する必要があります

### 問題4: データが書き込まれない

**原因:**
- シートが見つからない
- データの形式が正しくない

**対処法:**
1. **ログを確認**
   - 「📝 データを追加します...」が表示されているか確認
   - 「✅ データを追加しました」が表示されているか確認

2. **スプレッドシートを直接確認**
   - ログに表示されたスプレッドシートURLを開く
   - データが書き込まれているか確認

## 🔍 ログの検索方法

### ログパネル内で検索

1. **ログパネルを開く**
   - 実行履歴の行をクリックしてログを表示

2. **検索を実行**
   - `Ctrl+F`（Windows）/ `Cmd+F`（Mac）で検索
   - 例：「スプレッドシートURL」で検索

### フィルタを使用

1. **フィルタを追加**
   - 「+ フィルタを追加」ボタンをクリック
   - 関数名（例：`doPost`）でフィルタ

2. **日付範囲でフィルタ**
   - 特定の日付範囲の実行のみを表示

---

## 📝 ログのコピー方法

1. **ログパネルを開く**
   - 実行履歴の行をクリックしてログを表示

2. **ログを選択**
   - ログパネル内のテキストを選択（ドラッグ）

3. **コピー**
   - `Ctrl+C`（Windows）/ `Cmd+C`（Mac）でコピー

4. **貼り付け**
   - テキストエディタやメモ帳に`Ctrl+V`（Windows）/ `Cmd+V`（Mac）で貼り付け

## 💡 よくある質問

### Q1: 実行をクリックしてもログが表示されない

**原因と対処法:**

1. **方法3（コードエディタから）を試す（最も確実）**
   - 左側のメニューから「コード」をクリック
   - 「表示」→「ログ」を選択
   - 画面下部にログパネルが表示されます

2. **実行履歴の右端のメニューから開く**
   - 実行履歴の行の右端にある「⋮」アイコンをクリック
   - 「ログを表示」や「詳細を表示」を選択

3. **ブラウザの画面サイズを調整**
   - ログパネルが画面外に表示されている可能性があります
   - ブラウザのウィンドウサイズを調整してみます

4. **別のブラウザで試す**
   - 現在のブラウザで表示されない場合、別のブラウザで試してみます
   - Chrome、Firefox、Edgeなど

### Q2: クライアント側のログは取れているのに、Google Apps Script側のログが表示されない

**原因:**
- リクエストは送信されているが、Google Apps Script側でエラーが発生している
- デプロイが正しく行われていない

**対処法:**
1. Google Apps Scriptの実行数画面で最新の実行を確認
2. エラーメッセージがあるか確認
3. デプロイを再実行
4. 方法3（コードエディタから）でログを確認

### Q3: ログが多すぎて見づらい

**対処法:**
1. ログパネル内で`Ctrl+F`で検索
   - 例：「スプレッドシートURL」で検索
2. フィルタを使用して特定の関数のみを表示

### Q4: リアルタイムでログを確認したい

**対処法:**
1. 「リアルタイムで表示:」トグルをONにする
2. フォームを送信
3. 新しい実行が自動的に表示される
4. 最新の実行をクリックしてログを確認

## 📸 次のステップ（ログが表示されない場合の確実な方法）

### 方法A: コードエディタからログを確認（推奨）

1. **Google Apps Scriptのエディタを開く**
   - https://script.google.com/ にアクセス
   - プロジェクト「iTee予約管理」を開く

2. **コードエディタ画面に移動**
   - 左側のメニューから「コード」アイコン（</>）をクリック
   - または、上部の「コード」タブをクリック

3. **ログを表示**
   - メニューバーの「表示」をクリック
   - 「ログ」を選択
   - または、キーボードショートカット：`Ctrl+Enter`（Windows）/ `Cmd+Enter`（Mac）

4. **ログを確認**
   - 画面下部にログパネルが表示されます
   - 最新の実行ログが自動的に表示されます
   - 「📄 スプレッドシートURL」を探す
   - URLをクリックしてスプレッドシートを開く

5. **スプレッドシートを確認**
   - データが書き込まれているか確認
   - 「EmailHistory」シートを確認

### 方法B: 実行数画面からログを確認（通常の方法）

1. **Google Apps Scriptの実行数画面を開く**
   - https://script.google.com/ にアクセス
   - プロジェクト「iTee予約管理」を開く
   - 「実行数」をクリック

2. **最新の`doPost`実行をクリック**
   - 実行履歴のテーブルから最新の`doPost`実行を探す
   - その行をクリック

3. **ログパネルを確認**
   - 画面下部または右側にログパネルが表示されます
   - 表示されない場合は、方法Aを試してください

4. **ログを確認**
   - ログパネルに表示された内容を確認
   - 「📄 スプレッドシートURL」を探す
   - URLをクリックしてスプレッドシートを開く
