# saveEmailHistory関数の動作確認方法

`saveEmailHistory`関数が正しく動作しているか確認する方法を説明します。

---

## 🔍 確認方法

### 方法1: ブラウザのコンソールでテスト関数を実行（推奨）

1. **予約フォームページを開く**
   - https://www.pentech.info/school/mukonoso/html/reserve.html を開く
   - または、ローカルで`reserve.html`を開く

2. **開発者ツールを開く**
   - `F12`キーを押す
   - または、右クリック → 「検証」を選択

3. **Consoleタブを開く**
   - 「Console」タブを選択

4. **テスト関数を実行**
   - 以下のコマンドを入力してEnterキーを押す：
   ```javascript
   testSaveEmailHistory()
   ```

5. **結果を確認**
   - コンソールに以下のようなログが表示されます：
     ```
     🧪 saveEmailHistory関数のテストを開始します...
     📝 テスト1: 予約フォームの履歴を保存
     📧 メール送信履歴を保存しました: {id: "...", timestamp: "...", type: "reservation", data: {...}}
     結果: ✅ 成功
     📝 テスト2: お問い合わせフォームの履歴を保存
     📧 メール送信履歴を保存しました: {id: "...", timestamp: "...", type: "contact", data: {...}}
     結果: ✅ 成功
     📝 テスト3: 履歴を取得して確認
     履歴件数: 2
     履歴内容: [...]
     ```

### 方法2: 実際にフォームを送信して確認

1. **予約フォームを送信**
   - 予約フォームにテストデータを入力
   - 「予約を確定する」ボタンをクリック

2. **開発者ツールで確認**
   - `F12`キーで開発者ツールを開く
   - 「Console」タブで以下のログを確認：
     ```
     📧 メール送信履歴を保存しました: {id: "...", timestamp: "...", type: "reservation", data: {...}}
     ```

3. **履歴を確認**
   - コンソールで以下のコマンドを実行：
     ```javascript
     getEmailHistory()
     ```
   - 保存された履歴が表示されます

### 方法3: ローカルストレージを直接確認

1. **開発者ツールを開く**
   - `F12`キーを押す

2. **Applicationタブ（またはストレージタブ）を開く**
   - 「Application」タブを選択
   - 左側のメニューから「Local Storage」を展開
   - 現在のドメイン（例：`https://www.pentech.info`）をクリック

3. **`email_history`キーを確認**
   - `email_history`キーをクリック
   - 値が表示されます（JSON形式）

4. **値を確認**
   - 値はJSON形式の文字列です
   - コピーしてJSONビューアーで確認できます

## 🧪 テスト関数の使い方


2. お問い合わせフォームの履歴を保存

**期待される結果:**
- すべてのテストが「✅ 成功」と表示される
- 履歴件数が2件になる
- 最新の履歴が正しく保存されている

```javascript

```

**テスト内容:**
1. 現在の履歴を確認
2. 履歴をクリア
3. 新しい履歴を保存
4. 保存後の履歴を確認

**期待される結果:**
- クリア前の履歴件数が表示される
- クリア後の履歴件数が0になる
- 保存後の履歴件数が1になる

## ❌ よくある問題と解決方法

### 問題1: 「saveEmailHistory is not defined」エラー

**原因:**
- `email-history.js`が読み込まれていない
- スクリプトの読み込み順序が間違っている

**解決方法:**
1. `reserve.html`の12行目に`<script src="email-history.js"></script>`があるか確認
2. `config.js`の後に読み込まれているか確認
3. ブラウザをリロード（`Ctrl+F5`で強制リロード）

### 問題2: 履歴が保存されない

**原因:**
- ローカルストレージが無効になっている
- プライベートモード（シークレットモード）で開いている
- ブラウザの設定でローカルストレージがブロックされている

**解決方法:**
1. 通常モードでブラウザを開く
2. ブラウザの設定でローカルストレージが有効になっているか確認
3. 別のブラウザで試す

### 問題3: 履歴が表示されない

**原因:**
- `getEmailHistory()`関数が正しく動作していない
- ローカルストレージのデータが破損している

**解決方法:**
1. コンソールで`getEmailHistory()`を実行
2. エラーメッセージを確認
3. 必要に応じて`clearEmailHistory()`で履歴をクリアして再試行

## 💡 デバッグ用コマンド

ブラウザのコンソールで以下のコマンドを実行できます：

```javascript
// 履歴を取得
getEmailHistory()

// 履歴をクリア
clearEmailHistory()

// 履歴を保存（テスト用）
saveEmailHistory('reservation', {
    child_name: 'テスト',
    phone: '09012345678',
    email: 'test@example.com',
    school_type: '小学生',
    grade: '小学3年生',
    date: '2026/2/25 (水)',
    time: '8:10',
    message: 'テストメッセージ'
})

// ローカルストレージを直接確認
localStorage.getItem('email_history')

// ローカルストレージを直接削除
localStorage.removeItem('email_history')
```
