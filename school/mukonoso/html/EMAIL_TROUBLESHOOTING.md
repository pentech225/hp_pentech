# メールが届かない場合のトラブルシューティング

メールが届かない場合の確認手順と対処法を説明します。

---

## 🔍 確認手順

### ステップ1: Google Apps Scriptの実行ログを確認

**最も重要：まずここを確認してください。**

1. **Google Apps Scriptのエディタを開く**
   - https://script.google.com/ にアクセス
   - プロジェクト「iTee予約管理」を開く

2. **「実行数」をクリック**
   - 左側のメニューから「実行数」アイコンをクリック

3. **最新の`doPost`実行をクリック**
   - 実行履歴のテーブルから最新の`doPost`実行を探す
   - その行をクリック

4. **ログを確認**
   - 画面下部または右側にログパネルが表示されます
   - 以下のログを確認：

#### ✅ 成功時のログ例

```
=== doPost関数が呼び出されました ===
📥 受信データ: {...}
📅 予約フォームの処理を開始
📧 メール送信準備完了
📧 送信先: iteen.mukonosou@gmail.com
📧 件名: 無料体験予約のお申し込み
📧 メール送信を開始します...
✅ 管理者へのメール送信成功
✅ 送信先: iteen.mukonosou@gmail.com
✅ 件名: 無料体験予約のお申し込み
```

#### ❌ 失敗時のログ例

```
❌ 管理者へのメール送信エラー: Exception: メール送信に失敗しました
❌ エラーメッセージ: メール送信に失敗しました
❌ 送信先: iteen.mukonosou@gmail.com
❌ 件名: 無料体験予約のお申し込み
```

---

### ステップ2: Gmailの受信トレイを確認

1. **Gmailにログイン**
   - `iteen.mukonosou@gmail.com`でログイン
   - https://mail.google.com/ にアクセス

2. **受信トレイを確認**
   - 件名「無料体験予約のお申し込み」のメールを探す
   - 送信日時を確認（フォーム送信直後）

3. **迷惑メールフォルダを確認**
   - 左側のメニューから「迷惑メール」をクリック
   - メールが誤って迷惑メールに分類されていないか確認

4. **検索機能を使用**
   - Gmailの検索バーに「無料体験予約」と入力
   - 過去のメールも含めて検索

---

### ステップ3: Googleカレンダーを確認

メールが届かなくても、カレンダーにイベントが追加されていれば、リクエストはGoogle Apps Scriptに到達しています。

1. **Googleカレンダーを開く**
   - https://calendar.google.com/ にアクセス
   - `iteen.mukonosou@gmail.com`でログイン

2. **イベントを確認**
   - 「無料体験予約」というイベントが追加されているか確認
   - フォーム送信した日時と一致しているか確認

**カレンダーにイベントが追加されている場合：**
- ✅ リクエストはGoogle Apps Scriptに到達しています
- ✅ カレンダー追加は成功しています
- ❌ メール送信のみ失敗している可能性があります

## ❌ よくある問題と解決方法

### 問題1: ログに「❌ 管理者へのメール送信エラー」が表示される

#### 原因1: Gmail送信の権限が不足している

**エラーメッセージ例：**
```
Exception: このスクリプトには Gmail へのアクセス権限がありません。
```

**解決方法：**
1. Google Apps Scriptのエディタで「実行」→「doPost」を選択
2. 「承認が必要です」という警告が表示されたら、「承認」をクリック
3. Googleアカウントを選択
4. 「詳細」→「（安全ではないページ）に移動」をクリック
5. 「許可」をクリックしてGmail送信の権限を付与

#### 原因2: Gmailの送信制限に達している

**エラーメッセージ例：**
```
Exception: 1日の送信上限に達しました。
```

**解決方法：**
- Gmailの1日の送信上限（通常500通）に達している可能性があります
- 時間をおいて再試行してください
- 必要に応じて、送信元のGmailアカウントを変更してください

#### 原因3: メールアドレスの形式が正しくない

**エラーメッセージ例：**
```
Exception: メールアドレスの形式が正しくありません。
```

**解決方法：**
- `google-apps-script-code.js`の`to`変数を確認
- デフォルトは`iteen.mukonosou@gmail.com`です
- 正しいメールアドレス形式か確認してください

---

### 問題2: ログに「✅ 管理者へのメール送信成功」が表示されるが、メールが届かない

#### 原因1: スパムフォルダに入っている

**解決方法：**
1. Gmailの「迷惑メール」フォルダを確認
2. メールが見つかった場合：
   - メールを開く
   - 「迷惑メールではない」をクリック
   - 今後、同じ送信元からのメールが迷惑メールに入らないように設定

#### 原因2: メールフィルタで自動削除されている

**解決方法：**
1. Gmailの設定（⚙️アイコン）→「すべての設定を表示」を開く
2. 「フィルタとブロック中のアドレス」タブを開く
3. 該当するフィルタがないか確認
4. 必要に応じてフィルタを削除または修正

#### 原因3: 送信元アドレスが間違っている

**確認方法：**
1. Google Apps Scriptのログで「📧 送信先:」を確認
2. 正しいメールアドレス（`iteen.mukonosou@gmail.com`）になっているか確認

---

### 問題3: ログに「=== doPost関数が呼び出されました ===」が表示されない

**原因：**
- リクエストがGoogle Apps Scriptに到達していない
- デプロイが正しく行われていない

**解決方法：**
1. **デプロイを確認**
   - 「デプロイ」→「管理デプロイ」を開く
   - 最新のバージョンがデプロイされているか確認

2. **URLを確認**
   - `reserve.html`の`GOOGLE_APPS_SCRIPT_URL`が正しいか確認
   - デプロイURLと一致しているか確認

3. **ブラウザのコンソールを確認**
   - F12キーで開発者ツールを開く
   - 「Console」タブでエラーメッセージを確認
   - 「リクエストを送信しました (no-corsモード)」が表示されているか確認

---

### 問題4: カレンダーにイベントが追加されていない

**原因：**
- カレンダーIDが間違っている
- カレンダーへのアクセス権限がない

**解決方法：**
1. **カレンダーIDを確認**
   - `google-apps-script-code.js`の`CALENDAR_ID`を確認
   - Googleカレンダーの設定からカレンダーIDを再取得

2. **カレンダーの共有設定を確認**
   - Googleカレンダーで該当のカレンダーを開く
   - 「設定と共有」を開く
   - `iteen.mukonosou@gmail.com`に編集権限があるか確認

## 🔧 デバッグ用のテスト関数

Google Apps Scriptのエディタで、以下の関数を実行してメール送信をテストできます：

```javascript
// テスト用関数（メール送信のみ）
function testEmailSending() {
  try {
    const to = 'iteen.mukonosou@gmail.com';
    const subject = 'テストメール送信';
    const body = 'これはテストメールです。\n\nGoogle Apps Scriptから送信されました。';
    
    Logger.log('📧 テストメール送信を開始します...');
    Logger.log('📧 送信先: ' + to);
    Logger.log('📧 件名: ' + subject);
    
    GmailApp.sendEmail(to, subject, body);
    
    Logger.log('✅ テストメール送信成功');
  } catch (error) {
    Logger.log('❌ テストメール送信エラー: ' + error.toString());
    Logger.log('❌ エラーメッセージ: ' + error.message);
    Logger.log('❌ エラースタック: ' + error.stack);
  }
}
```

この関数を実行すると、メール送信のみをテストできます。
