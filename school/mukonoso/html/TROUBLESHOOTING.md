## Google Apps Script設定手順（詳細版）

### 全体の流れ

1. Google Apps Scriptプロジェクトを作成
2. コードを貼り付け
3. テスト実行（権限承認）
4. Webアプリとしてデプロイ
5. URLを取得してHTMLに設定

---

### ステップ1: Google Apps Scriptプロジェクトの作成

1. ブラウザで https://script.google.com/ を開く
2. `iteen.mukonosou@gmail.com` のアカウントでログイン
3. 左上の「新しいプロジェクト」をクリック
4. 左上の「無題のプロジェクト」をクリック
5. 「iTeen 問い合わせ・予約フォーム」などに変更して、Enterキーで確定

---

### ステップ2: コードを貼り付け

1. エディタに表示されている `function myFunction() {}` を削除
2. `google-apps-script-code.js` ファイルを開く
3. すべてのコードをコピー（Ctrl+A → Ctrl+C）
4. Google Apps Scriptのエディタに貼り付け（Ctrl+V）
5. 左上の「保存」ボタンをクリック（または Ctrl+S）

---

### ステップ3: テスト実行（権限承認）

#### 問い合わせフォームのテスト

1. エディタ上部の関数選択ドロップダウンから「`testContactForm`」を選択
2. 上部の「実行」ボタン（▶️）をクリック
3. 「承認が必要です」という警告が表示されたら「承認」をクリック
4. Googleアカウントを選択
5. 「詳細」→「iTeen 問い合わせ・予約フォーム（安全ではないページ）に移動」をクリック
6. 「許可」をクリック
7. 下部の「実行ログ」タブで「成功」と表示されることを確認
8. `iteen.mukonosou@gmail.com`の受信トレイにメールが届いているか確認

#### 予約フォームのテスト

1. 関数選択ドロップダウンから「`testReservationForm`」を選択
2. 「実行」ボタンをクリック
3. 実行ログを確認
4. Googleカレンダーに予約が追加されたか確認

---

### ステップ4: Webアプリとしてデプロイ

1. 右上の「デプロイ」ボタンをクリック
2. 「新しいデプロイ」をクリック
3. 歯車アイコン（⚙️）をクリック
4. 「種類の選択」から「ウェブアプリ」を選択
5. 設定を入力：
   - **説明**: `iTeen 問い合わせ・予約フォーム v1`（任意）
   - **次のユーザーとして実行**: 「自分」を選択
   - **アクセスできるユーザー**: 「全員」を選択（**重要**）
6. 「デプロイ」ボタンをクリック
7. 「承認が必要です」が表示されたら「承認」をクリック → 権限を許可
8. 「WebアプリのURL」をコピーして保存
   - 例: `https://script.google.com/macros/s/AKfycbwBemLT.../exec`

---

### ステップ5: HTMLファイルにURLを設定

#### contact.html の設定

1. `school/mukonoso/html/contact.html` を開く
2. 509行目付近の`GOOGLE_APPS_SCRIPT_URL`を見つける
3. ステップ4でコピーしたURLに置き換える

#### reserve.html の設定

1. `school/mukonoso/html/reserve.html` を開く
2. `GOOGLE_APPS_SCRIPT_URL` を検索（Ctrl+F）
3. ステップ4でコピーしたURLに置き換える（contact.htmlと同じURLを使用）

---

### ステップ6: 動作確認

#### 問い合わせフォームの確認

1. `contact.html` をブラウザで開く
2. フォームに入力（メールアドレス、電話番号、お問い合わせ内容）
3. 「送信する」ボタンをクリック
4. 確認：
   - 成功モーダルが表示される
   - `iteen.mukonosou@gmail.com` にメールが届く
   - 入力したメールアドレスに自動応答メールが届く

#### 予約フォームの確認

1. `reserve.html` をブラウザで開く
2. 学校区別、学年、日時などを選択
3. 「予約を確定する」ボタンをクリック
4. 確認：
   - 成功メッセージが表示される
   - `iteen.mukonosou@gmail.com` にメールが届く
   - Googleカレンダーに予約が追加される

---

### コードを更新した場合の再デプロイ

1. コードを修正して保存
2. 「デプロイ」→「管理デプロイ」を開く
3. 歯車アイコン（⚙️）をクリック
4. 「新しいバージョン」を選択
5. 「デプロイ」をクリック
6. HTMLフォームから送信して動作確認

---

### 重要なポイント

1. **`doPost` は直接実行できない**
   - Webアプリとしてデプロイしてから、HTTP POSTリクエストで呼び出す必要がある
   - テストする場合は `testContactForm` または `testReservationForm` を使用

2. **アクセス権限は「全員」に設定**
   - これがないと、外部からアクセスできません

3. **初回実行時に権限を承認**
   - Gmail送信の権限
   - Googleカレンダーへのアクセス権限

4. **URLは同じものを使用**
   - 問い合わせフォームと予約フォームは同じGoogle Apps Scriptを使用
   - 同じURLを両方のHTMLファイルに設定

## Google Apps Script メール送信トラブルシューティングガイド

### 現在の状況

CORSエラーが発生していますが、これは**正常な動作**です。Google Apps ScriptはCORSを完全にはサポートしていないため、`no-cors`モードでフォールバックしています。

**重要なポイント：**
- リクエストはGoogle Apps Scriptに到達しています
- `no-cors`モードではレスポンスを取得できません
- メール送信の成功/失敗は、Google Apps Scriptの実行ログで確認する必要があります

---

### メール送信の確認方法

#### 1. Google Apps Scriptの実行ログを確認

1. [Google Apps Script](https://script.google.com/)にアクセス
2. プロジェクト「iTeen予約フォーム」を開く
3. 左側のメニューから「**実行数**」をクリック
4. 最新の実行ログを確認

**成功時のログ例：**
```
✅ 管理者へのメール送信成功
✅ 自動応答メール送信成功: hikosannsei@gmail.com
Google Sheetsに履歴を保存しました
```

**失敗時のログ例：**
```
❌ 管理者へのメール送信エラー: [エラー内容]
❌ 自動応答メール送信エラー: [エラー内容]
```

#### 2. Gmailの受信トレイを確認

1. `iteen.mukonosou@gmail.com`にログイン
2. 受信トレイを確認
3. 迷惑メールフォルダも確認

#### 3. Googleカレンダーを確認

1. [Googleカレンダー](https://calendar.google.com/)にアクセス
2. 「無料体験予約」というイベントが追加されているか確認

---

### よくある問題と解決方法

#### 問題1: メールが届かない

**確認事項：**
1. Google Apps Scriptの実行ログでエラーが発生していないか確認
2. Gmailの送信元アドレス（`iteen.mukonosou@gmail.com`）が正しいか確認
3. Gmailの送信制限に達していないか確認（1日あたりの送信上限）

**解決方法：**
- 実行ログにエラーが表示されている場合は、そのエラー内容を確認
- Gmailの送信制限に達している場合は、時間をおいて再試行

#### 問題2: CORSエラーが表示される

**これは正常な動作です。**
- Google Apps ScriptはCORSを完全にはサポートしていません
- `no-cors`モードでフォールバックしているため、エラーが表示されますが、リクエストは正常に送信されています
- メール送信の成功/失敗は、Google Apps Scriptの実行ログで確認してください

#### 問題3: 「Access denied」または「アクセスが拒否されました」

**原因**: アクセス権限が「全員」に設定されていない

**解決方法:**
1. 「デプロイ」→「管理デプロイ」を開く
2. 歯車アイコン（⚙️）をクリック
3. 「アクセスできるユーザー」を「全員」に変更
4. 「新しいバージョン」を作成して再デプロイ

#### 問題4: Google Apps Scriptの実行ログにエラーが表示される

##### 「Gmail送信の権限がありません」
- **原因**: Gmail送信の権限が付与されていない
- **解決方法**:
  1. Google Apps Scriptのエディタで「実行」→「doPost」を選択
  2. 「承認が必要です」という警告が表示されたら、「承認」をクリック
  3. Gmail送信の権限を付与

##### 「カレンダーが見つかりません」
- **原因**: カレンダーIDが間違っている、またはカレンダーへのアクセス権限がない
- **解決方法**:
  1. `google-apps-script-code.js`の`CALENDAR_ID`を確認
  2. Googleカレンダーの設定からカレンダーIDを再取得
  3. カレンダーの共有設定で、スクリプトを実行するアカウントに編集権限があるか確認

##### 「日付または時間が指定されていません」
- **原因**: フォームから送信されたデータに`date_raw`または`time`フィールドが含まれていない
- **解決方法**:
  1. ブラウザのコンソール（F12キー）で送信データを確認
  2. `date_raw`と`time`フィールドが正しく送信されているか確認

---

### デプロイ設定の確認方法

1. Google Apps Scriptのエディタで「**デプロイ**」→「**管理デプロイ**」をクリック
2. 最新のデプロイを選択
3. 以下の設定を確認：
   - **種類**: 「ウェブアプリ」になっているか
   - **アクセスできるユーザー**: 「**全員**」に設定されているか（重要）
   - **次のユーザーとして実行**: 「自分」に設定されているか

---

### コードを更新した場合の再デプロイ

1. Google Apps Scriptのエディタでコードを更新
2. 「保存」ボタンをクリック
3. 「**デプロイ**」→「**新しいデプロイ**」をクリック
4. 歯車アイコン（⚙️）をクリック
5. 「種類の選択」→「ウェブアプリ」を選択
6. 設定を確認：
   - **次のユーザーとして実行**: 「自分」
   - **アクセスできるユーザー**: 「**全員**」（重要）
7. 「デプロイ」をクリック
8. **新しいWebアプリのURL**をコピー
9. `config.js`の`GOOGLE_APPS_SCRIPT_URL`を更新

```javascript
function testReservationForm() {

    date: '2026/2/13(金)',
    time: '17:10',
    date_raw: '2026-02-13',
    school_type: '公立',
    grade: '小学3年生',
    child_name: 'テスト太郎',
    phone: '09012345678',
    email: 'test@example.com',
    message: 'テストメッセージ'
  };

  const mockEvent = {
    postData: {
      contents: JSON.stringify(testData)
    }
  };

  const result = doPost(mockEvent);
  Logger.log(result.getContent());
}
```

### よくあるエラーまとめ

| エラー | 原因 | 解決方法 |
|--------|------|----------|
| "Access denied" | アクセス権限が「全員」に設定されていない | デプロイ設定を「全員」に変更 |
| "Script function not found" | 関数名が正しくない | `doPost`であることを確認 |
| "CORS error" | 正常な動作 | `no-cors`モードのため問題なし。ログで確認 |
| "リクエストデータが正しくありません" | `doPost`を直接実行しようとしている | テスト関数を使用 |
| Gmail送信権限エラー | 権限が付与されていない | 承認画面で権限を付与 |
| カレンダーが見つからない | カレンダーIDが間違っている | カレンダーIDを再取得して確認 |

### サポート

1. Google Apps Scriptの実行ログ

3. メールの受信トレイ
4. Googleカレンダーのイベント一覧

2. コードを更新した後
3. エラーが発生した時
4. 定期的なメンテナンス時（月1回など）
