# Google Sheetsへのアクセス権限について

## 結論

**基本的には、Google Sheets側でアクセス権限を調整する必要はありません。**

ただし、既存のスプレッドシートを使用する場合（2回目以降）は、Google Apps Scriptを実行しているアカウントがそのスプレッドシートにアクセスできる必要があります。

---

## アクセス権限が必要なケースと不要なケース

### ✅ アクセス権限の調整が不要なケース

#### ケース1: 初回実行時（新規作成）

```javascript
if (!spreadsheetId) {
  spreadsheet = SpreadsheetApp.create('iTeen 武庫之荘校 - メール送信履歴');
  // ...
}
```

**理由:**
- `SpreadsheetApp.create()`で新規作成する場合、作成したアカウント（iteen.mukonosou@gmail.com）が所有者になる
- 所有者は自動的にすべての権限を持っているため、アクセス権限の問題は発生しない

#### ケース2: エラー時に新規作成

```javascript
} catch (openError) {
  // エラーの場合は新規作成
  spreadsheet = SpreadsheetApp.create('iTeen 武庫之荘校 - メール送信履歴');
  // ...
}
```

**理由:**
- 既存のスプレッドシートを開けない場合、新しいスプレッドシートを作成する
- 新規作成なので、作成したアカウントが所有者になる

---

### ⚠️ アクセス権限の確認が必要なケース

#### ケース: 既存のスプレッドシートを使用する場合（2回目以降）

```javascript
} else {
  spreadsheet = SpreadsheetApp.openById(spreadsheetId);
  // ...
}
```

**必要な条件:**
- Google Apps Scriptを実行しているアカウント（iteen.mukonosou@gmail.com）が、そのスプレッドシートにアクセスできる必要がある
- スプレッドシートの所有者、または編集者/閲覧者の権限が必要

---

## Google Apps Scriptの実行アカウント

### 「次のユーザーとして実行」の設定

Google Apps Scriptのデプロイ設定で「次のユーザーとして実行」が「自分」に設定されている場合：

- **実行アカウント**: `iteen.mukonosou@gmail.com`
- **権限**: このアカウントの権限でGoogle Sheetsにアクセスする

### スプレッドシートの作成者

- **初回実行時**: `iteen.mukonosou@gmail.com`が所有者として作成
- **結果**: このアカウントは自動的にすべての権限を持っている

---

## アクセス権限エラーが発生する場合

### エラーの原因

1. **既存のスプレッドシートが別のアカウントで作成された**
   - スクリプトプロパティに保存されているスプレッドシートIDが、別のアカウントで作成されたもの
   - `iteen.mukonosou@gmail.com`がそのスプレッドシートにアクセスできない

2. **スプレッドシートが削除された**
   - スクリプトプロパティに保存されているスプレッドシートIDのスプレッドシートが存在しない

3. **スプレッドシートの共有設定が変更された**
   - 以前はアクセスできたが、共有設定が変更されてアクセスできなくなった

### 現在のコードの動作

```javascript
} catch (openError) {
  // エラーの場合は新規作成
  spreadsheet = SpreadsheetApp.create('iTeen 武庫之荘校 - メール送信履歴');
  // 新しいスプレッドシートIDをスクリプトプロパティに保存
  properties.setProperty('GOOGLE_SHEETS_SPREADSHEET_ID', spreadsheetId);
}
```

**動作:**
- 既存のスプレッドシートを開けない場合、自動的に新しいスプレッドシートを作成
- 新しいスプレッドシートIDをスクリプトプロパティに保存
- 次回以降は新しいスプレッドシートを使用

**結果:**
- アクセス権限エラーが発生しても、自動的に新しいスプレッドシートが作成されるため、問題は解決される

---

## アクセス権限を調整する必要がある場合

### 既存のスプレッドシートを使用したい場合

もし、特定の既存のスプレッドシートを使用したい場合は、以下の手順でアクセス権限を設定してください：

#### ステップ1: スプレッドシートの共有設定を確認

1. [Google Sheets](https://sheets.google.com)にアクセス
2. 使用したいスプレッドシートを開く
3. 右上の「共有」ボタンをクリック

#### ステップ2: Google Apps Scriptの実行アカウントに権限を付与

1. 「ユーザーやグループを追加」に`iteen.mukonosou@gmail.com`を入力
2. 権限を「編集者」または「閲覧者」に設定
3. 「送信」をクリック

#### ステップ3: スクリプトプロパティにスプレッドシートIDを設定

1. Google Apps Scriptのエディタで「プロジェクトの設定」（⚙️）をクリック
2. 「スクリプト プロパティ」セクションで`GOOGLE_SHEETS_SPREADSHEET_ID`を設定
3. スプレッドシートのURLからIDを取得：
   - 例: `https://docs.google.com/spreadsheets/d/1a2b3c4d5e6f7g8h9i0j/edit`
   - ID: `1a2b3c4d5e6f7g8h9i0j`

## 推奨される設定

### 推奨: 自動的に新規作成される設定（現在の設定）

**メリット:**
- アクセス権限の調整が不要
- エラーが発生しても自動的に新しいスプレッドシートが作成される
- 管理が簡単

**デメリット:**
- エラー時に新しいスプレッドシートが作成されるため、データが分散する可能性がある

### 代替案: 手動でスプレッドシートを作成して共有設定

**メリット:**
- 特定のスプレッドシートを確実に使用できる
- データが一箇所に集まる

**デメリット:**
- アクセス権限の設定が必要
- 共有設定を忘れるとエラーが発生する

## まとめ

### 現在の設定では

1. **初回実行時**: 新しいスプレッドシートが自動的に作成される（アクセス権限の調整不要）
2. **2回目以降**: スクリプトプロパティからスプレッドシートIDを取得して開く
3. **エラー時**: 新しいスプレッドシートが自動的に作成される（アクセス権限の調整不要）

### アクセス権限の調整が必要な場合

- 既存のスプレッドシートを確実に使用したい場合のみ
- そのスプレッドシートに`iteen.mukonosou@gmail.com`を共有する必要がある

### 現在のコードの動作

- アクセス権限エラーが発生しても、自動的に新しいスプレッドシートが作成される
- 基本的には、Google Sheets側でアクセス権限を調整する必要はない

---

## 確認方法

### スプレッドシートが正しく作成されているか確認

1. Google Apps Scriptのログで、スプレッドシートURLを確認
2. そのURLを開いて、スプレッドシートが存在するか確認
3. スプレッドシートの所有者が`iteen.mukonosou@gmail.com`か確認

### アクセス権限エラーが発生しているか確認

1. Google Apps Scriptのログで、以下のメッセージを確認：
   - `❌ スプレッドシートを開くエラー:`
   - `⚠️ アクセス権限エラーの可能性があります`
2. これらのメッセージが表示されている場合、既存のスプレッドシートにアクセスできない
3. コードが自動的に新しいスプレッドシートを作成するため、問題は解決される
