# Cloud のログが見れない理由と対処法

「Cloud のログ」が見れない場合の原因と対処法を説明します。

---

## 🔍 ログの種類

Google Apps Scriptには2種類のログがあります：

### 1. `Logger.log()` - エディタ実行時のログ

- **表示場所**: 実行数画面の個別の実行をクリック
- **確認方法**: 実行履歴の行をクリックすると、画面下部または右側にログパネルが表示される
- **使用例**: エディタから直接実行した場合

### 2. `console.log()` - Webアプリ実行時のログ（Cloud Logging）

- **表示場所**: Cloud Logging（別のサービス）
- **確認方法**: 「Cloud のログ」メニューから確認（有効化が必要な場合あり）
- **使用例**: デプロイされたWebアプリとして実行された場合

---

## ❌ 「Cloud のログ」が見れない理由

### 理由1: Cloud Logging APIが有効になっていない

**原因:**
- Google Cloud Platform（GCP）のCloud Logging APIが有効になっていない
- デフォルトでは、Google Apps Scriptのログは実行数画面で確認できます

**対処法:**
- 通常は、実行数画面で個別の実行をクリックしてログを確認すれば十分です
- Cloud Loggingが必要な場合は、以下の手順で有効化できます：
  
  ### Cloud Logging APIの有効化手順
  
  1. **Google Apps Scriptのプロジェクト設定を開く**
     - 左側のメニューから「プロジェクトの設定」（⚙️アイコン）をクリック
     - 「Google Cloud Platform（GCP）プロジェクト」セクションを確認
     - プロジェクトIDが表示されているか確認
     - プロジェクトIDがない場合は、「プロジェクト番号を変更」をクリックして新しいプロジェクトを作成
       - **推奨**: 「こちら」リンクから新しいプロジェクトを作成し、自動的に割り当てられたプロジェクト番号を使用
       - **プロジェクト名**: `iTee予約管理` または `iteen-reservation-management`（わかりやすい名前）
       - **プロジェクト番号**: 自動的に割り当てられるため、手動で設定する必要はありません
  
  2. **Google Cloud Consoleを開く**
     - https://console.cloud.google.com/ にアクセス
     - Google Apps Scriptで確認したGCPプロジェクトを選択（画面左上のプロジェクト選択ドロップダウンから）
  
  3. **Cloud Logging APIを有効化**
     - 左側のメニューから「APIとサービス」→「ライブラリ」をクリック
     - 検索バーに「Cloud Logging API」と入力
     - 「Cloud Logging API」をクリック
     - 「有効にする」ボタンをクリック
     - 有効化が完了するまで数秒待つ
  
  4. **ログを確認**
     - Google Cloud Consoleで「ロギング」→「ログエクスプローラー」を開く
     - リソースタイプで「Google Apps Script」を選択
     - ログを確認
  
  **注意:**
  - Cloud Logging APIの有効化には、Google Cloud Platform（GCP）のアカウントが必要です
  - 無料枠内であれば、追加費用はかかりません
  - 通常のデバッグには、実行数画面でログを確認すれば十分です

### 理由2: ログがまだ生成されていない

**原因:**
- デプロイされたWebアプリとして実行されていない
- `console.log()`が実行されていない

**対処法:**
- 本番環境で予約フォームを送信
- 実行数画面で最新の`doPost`実行をクリックしてログを確認

### 理由3: 権限の問題

**原因:**
- Cloud Loggingにアクセスする権限がない
- GCPプロジェクトへのアクセス権限がない

**対処法:**
- 通常は、実行数画面でログを確認すれば十分です
- Cloud Loggingが必要な場合は、GCPプロジェクトの設定を確認

## ✅ 推奨されるログ確認方法

### 方法1: 実行数画面で個別の実行をクリック（最も簡単）

1. **実行数画面を開く**
   - 左側のメニューから「実行数」をクリック

2. **最新の`doPost`実行をクリック**
   - 実行履歴のテーブルから最新の`doPost`実行を探す
   - その行をクリック

3. **ログパネルを確認**
   - 画面下部または右側にログパネルが表示されます
   - `Logger.log()`と`console.log()`の両方が表示されます

**この方法の利点:**
- 追加の設定が不要
- すべてのログが表示される
- エラーメッセージも確認できる

### 方法2: コードエディタ画面からログを確認（確実に表示される）

1. **コードエディタ画面に移動**
   - 左側のメニューから「コード」アイコン（</>）をクリック

2. **「表示」メニューからログを開く**
   - メニューバーの「表示」をクリック
   - 「ログ」を選択
   - または、キーボードショートカット：`Ctrl+Enter`（Windows）/ `Cmd+Enter`（Mac）

3. **ログパネルを確認**
   - 画面下部にログパネルが表示されます
   - 最新の実行ログが自動的に表示されます

## 🔧 Cloud Loggingを有効化する場合（上級者向け）

Cloud Loggingが必要な場合（通常は不要）：

### ステップ1: Google Cloud Platform（GCP）プロジェクトを確認

1. **Google Apps Scriptのプロジェクト設定を開く**
   - 左側のメニューから「プロジェクトの設定」（⚙️アイコン）をクリック

2. **Google Cloud Platform（GCP）プロジェクトを確認**
   - 「Google Cloud Platform（GCP）プロジェクト」セクションを確認
   - プロジェクトIDが表示されているか確認
   
   **既存のプロジェクトがある場合:**
   - プロジェクトIDが表示されている場合は、そのまま使用できます
   - プロジェクト番号は自動的に割り当てられているため、変更する必要はありません
   
   **新しいプロジェクトを作成する場合:**
   - 「プロジェクト番号を変更」をクリック
   - 「こちら」リンクをクリックしてGoogle Cloud Consoleを開く
   - 新しいプロジェクトを作成
     - **プロジェクト名**: `iTee予約管理` または `iteen-reservation-management`（わかりやすい名前）
     - **プロジェクトID**: 自動生成されるか、手動で設定（例：`iteen-reservation-management`）
     - **プロジェクト番号**: 自動的に割り当てられる（手動では設定不可）
   - 作成後、Google Apps Scriptに戻り、自動的に割り当てられたプロジェクト番号を入力
   - 「プロジェクトを設定」ボタンをクリック

### ステップ2: Cloud Logging APIを有効化

1. **Google Cloud Consoleを開く**
   - https://console.cloud.google.com/ にアクセス
   - 画面左上のプロジェクト選択ドロップダウンから、Google Apps Scriptで確認したGCPプロジェクトを選択
   - プロジェクトが表示されていない場合は、Google Apps Scriptのプロジェクト設定で「プロジェクト番号を変更」をクリックして新しいプロジェクトを作成

2. **APIとサービスライブラリを開く**
   - 左側のメニューから「APIとサービス」をクリック
   - 「ライブラリ」をクリック

3. **Cloud Logging APIを検索**
   - 検索バーに「Cloud Logging API」と入力
   - 検索結果から「Cloud Logging API」をクリック

4. **APIを有効化**
   - 「有効にする」ボタンをクリック
   - 有効化が完了するまで数秒〜数分待つ
   - 「APIが有効になりました」というメッセージが表示されれば成功

5. **確認**
   - 「APIとサービス」→「有効なAPI」を開く
   - 「Cloud Logging API」がリストに表示されているか確認

### ステップ3: ログエクスプローラーを開く

1. **左側のナビゲーションメニューから「ロギング」を選択**
   - 現在は「API とサービス」セクションが選択されています
   - 左側のメニューをスクロールして「ロギング」を探す
   - 「ロギング」をクリック

2. **「ログエクスプローラー」をクリック**
   - 「ロギング」セクションを展開すると、以下のサブメニューが表示されます：
     - 「ログエクスプローラー」（Logs Explorer）
     - 「ログベースの指標」（Log-based metrics）
     - 「エクスポート」（Exports）
     - など
   - 「ログエクスプローラー」をクリック

3. **ログをフィルタ**
   - 「リソースタイプ」で「Google Apps Script」を選択
   - 必要に応じて、時間範囲やログレベルでフィルタ
   - 「クエリを実行」ボタンをクリック

4. **ログを確認**
   - ログが表示されます
   - `console.log()`で出力した内容が表示されます

**注意:**
- ログエクスプローラーは、左側のナビゲーションメニューから「ロギング」→「ログエクスプローラー」でアクセスできます
- 現在の画面（API / サービスの詳細）からは直接アクセスできません
- 左側のメニューから「ロギング」セクションに移動する必要があります

**注意:**
- Cloud Logging APIの有効化には、Google Cloud Platform（GCP）のアカウントが必要です
- 無料枠内であれば、追加費用はかかりません（月間50GBのログ保存まで無料）
- Cloud Loggingは通常、大規模な運用や詳細な分析が必要な場合に使用します
- 通常のデバッグには、実行数画面でログを確認すれば十分です

### トラブルシューティング

#### 問題1: 「APIとサービス」メニューが表示されない

**原因:**
- GCPプロジェクトが選択されていない
- 権限が不足している

**解決方法:**
1. 画面左上のプロジェクト選択ドロップダウンで、正しいプロジェクトが選択されているか確認
2. Google Apps Scriptのプロジェクト設定で、GCPプロジェクトIDを確認

#### 問題2: 「有効にする」ボタンが表示されない

**原因:**
- APIが既に有効になっている
- 権限が不足している

**解決方法:**
1. 「APIとサービス」→「有効なAPI」を開く
2. 「Cloud Logging API」がリストに表示されているか確認
3. 表示されている場合は、既に有効になっています

#### 問題3: ログが表示されない

**原因:**
- ログがまだ生成されていない
- フィルタが正しく設定されていない

**解決方法:**
1. 本番環境で予約フォームを送信
2. 時間範囲を広げる（例：過去1時間）
3. リソースタイプで「Google Apps Script」が選択されているか確認

## 💡 よくある質問

### Q1: 「Cloud のログ」は必要ですか？

**A:** 通常は不要です。実行数画面で個別の実行をクリックしてログを確認すれば十分です。

### Q2: `console.log()`のログはどこで確認できますか？

**A:** 実行数画面で個別の実行をクリックすると、`Logger.log()`と`console.log()`の両方が表示されます。

### Q3: Cloud Loggingを有効化する必要がありますか？

**A:** 通常は不要です。実行数画面でログを確認すれば十分です。Cloud Loggingは、大規模な運用や詳細な分析が必要な場合に使用します。
